1. Why do we need a viewport meta tag?

This tag is essential for a responsive web design. It controls the layout of our webpage by controlling its width and scale of the viewport. It allows users to still view a webpage easily on smaller screens like phones. Without it, websites may be displayed as a zoomed-out version of the desktop layout. Adding this tag ensures that our page scales correctly on different sizes of devices and allows CSS media queries to adapt to various screens.

2. When you select the same tag twice, and in one of them you say "flex-direction: column;" and the other says "flex-direction: row;", which one takes priority?

I learned in class that the rule that comes last in the CSS stylesheet will overwrite the first one. The browser will use the one that appears later in the CSS because it is the last command that we used. I also learned that when we do not want to deal with this, we can just add a !important rule, which makes whatever we put with it the priority.

3. How do you change the CSS for a webpage displayed on a phone?

We can change the CSS for mobile devices by using media query. For example, we did "@media (max-width: 600px) in this activity and we put all the changes we want to have and modified it in this media query.
