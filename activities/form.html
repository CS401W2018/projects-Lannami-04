1. The form tag has an "action" attribute. What does it do?

The action attribute tells the browser where to send the form data when it is submitted. It could send it to a server script or to another page that will handle the data and perform tasks like saving information to a database.

2. The form tag has a "method" attribute. What does it do?

The method attribute in the form tag defines how the form data will be sent to the server. The values we used in class were GET and POST.

3. What is the difference between a "POST" and a "GET"?

GET appends data to the URL, making it visible in the browser's address bar. It is best used for requests that do not change data on the server. POST, on the other hand, sends data inside the request body which is hidden from the URL. It supports larger data and our professor said we should use this more because it is better in handling login forms or sending personal information as it keeps the data more secure.
